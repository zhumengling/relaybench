# 阶段项目报告

## 当前阶段重点

- 为端口扫描提供独立工作区与交互入口。
- 参考常见开源扫描器的优点，落地本地内置扫描引擎。
- 补齐实时进度、结构化结果表格、自定义端口输入与日志输出。
- 与网络、路由、测速、中转站、STUN、IP / 分流等模块统一到同一套报告导出体系。

## 本阶段完成情况

| 项目 | 状态 | 说明 |
| --- | --- | --- |
| 端口扫描独立标签页 | 已完成 | 支持目标输入、模板选择、自定义端口、引擎检测、运行扫描。 |
| 内置本地扫描引擎 | 已完成 | 采用异步 TCP Connect + 有界并发，不依赖外部扫描器。 |
| 轻量服务识别 | 已完成 | 支持 Banner、TLS、HTTP、Redis 等轻量探测。 |
| 实时进度反馈 | 已完成 | 支持进度条、实时状态与逐步追加日志。 |
| 结构化结果表格 | 已完成 | 结果表格展示终端、协议、延迟、服务提示、Banner、TLS、应用摘要与备注。 |
| 状态持久化与兼容迁移 | 已完成 | 新状态字段已切换为 PortScan 命名，并兼容旧状态键迁移。 |
| 报告导出接入 | 已完成 | 结构化报告和原始输出已切换为 portScan 字段。 |

## 代码结构说明

- `NetTest.Core`：网络检测、ChatGPT Trace、STUN、测速、路由、中转站、IP / 分流与端口扫描核心服务。
- `NetTest.App`：WPF 桌面界面、ViewModel、状态持久化、报告导出、图表与地图渲染。
- `data`：本地状态、报告归档与运行缓存。

## 后续可继续增强的方向

- 增加 UDP 或更丰富的应用层探测能力。
- 引入批量目标扫描任务视图与历史趋势统计。
- 扩展端口模板体系，例如数据库、邮件、Kubernetes、DevOps 常见端口模板。
- 为结果表格增加筛选、导出 CSV、复制终端等交互。
