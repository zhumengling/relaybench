# RelayBench UI 重构实现计划

> 日期：2026-04-19  
> 项目路径：`H:\nettest`  
> 适用范围：`NetTest.App`、相关 ViewModel 与页面结构调整

## 1. 目标

本轮改造的目标是把原本偏“技术模块平铺”的桌面工具，重构为更符合实际使用流程的中转站测试工作台：

- 启动后直接进入“单站测试”；
- 用左侧导航替代旧的多标签页结构；
- 把“深度测试”并入单站与批量流程，而不是继续作为独立一级页面；
- 让“批量对比”遵循“入口组 → 快速对比 → 排行 → 手动勾选 → 深度测试”的固定流程；
- 让“网络复核”只承担辅助排查职责；
- 收紧字体、卡片和留白，形成白底 benchmark 风格的高密度工作台界面。

## 2. 已确认的产品决策

### 2.1 一级导航

一级导航固定为 4 项：

1. 单站测试
2. 批量对比
3. 网络复核
4. 历史报告

### 2.2 单站测试

- 默认模式为“快速测试”；
- 页面不再重复基础站点配置；
- 深度测试只作为单站页面中的一种测试模式存在。

### 2.3 批量对比

- 默认显示排行榜视图；
- 图表只负责展示，不承担勾选交互；
- 用户只能在排行榜列表项中手动勾选候选站点；
- 勾选后才允许执行批量深度测试。

### 2.4 网络复核

- 不再平铺所有网络功能；
- 先选异常类型，再推荐复核模块；
- 先展示结论与下一步建议，再展示原始明细。

## 3. 实现拆分

### 任务 1：导航与页面状态重构

涉及内容：

- 新增导航状态；
- 新增页面标题、副标题与可见性派生属性；
- 应用启动后默认落到“单站测试”；
- 只持久化必要的模式状态，不持久化当前导航页。

验证要点：

- 启动页固定为“单站测试”；
- 默认单站模式为“快速测试”；
- 默认网络复核异常类型合理。

### 任务 2：主窗口改造为左侧工作台结构

涉及内容：

- 抽离主题资源；
- 主窗口改为左侧导航 + 右侧主工作区；
- 页面级 UserControl 拆分；
- 统一主题、密度与卡片样式。

验证要点：

- 不再存在首页或总览页；
- 一级导航收敛为 4 项；
- 页面高度、卡片内边距与字体更紧凑。

### 任务 3：单站测试页重构

涉及内容：

- 形成“站点配置 / 模式参数与执行 / 结果输出”三段式结构；
- 快速测试、稳定性测试、深度测试三种模式统一收口；
- 深度测试的高级选项只在对应模式中显示。

验证要点：

- 默认进入快速测试；
- 站点配置不重复出现；
- 深度测试不会误调用快速测试路径。

### 任务 4：批量对比页重构

涉及内容：

- 批量入口组管理；
- 快速对比后的排行榜图表与列表；
- 排行列表项手动勾选；
- 对勾选站点发起批量深度测试；
- 批量图表与单站图表彻底分离。

验证要点：

- 快速对比后默认不勾选任何站点；
- 图表只读，列表可勾选；
- 勾选后才能触发批量深测；
- 批量深测不会覆盖单站当前配置。

### 任务 5：网络复核页重构

涉及内容：

- 用异常类型驱动页面组织；
- 将网络模块分为基础复核、性能复核、出口复核和高级复核；
- 优先展示结论、判断与推荐动作。

验证要点：

- 首屏优先展示建议与结论；
- 原始输出退到次级位置；
- 仍可触发全部已有网络诊断命令。

### 任务 6：历史报告与术语统一

涉及内容：

- 整理历史报告页面；
- 统一“单站测试 / 批量对比 / 网络复核 / 历史报告”等术语；
- 调整导出与历史摘要文案；
- 做一轮全局密度收尾。

验证要点：

- 历史页只负责回看与导出；
- 新生成的记录和报告术语一致；
- 全局视觉密度明显收紧。

## 6. 风险与控制点

- `MainWindow.xaml` 改动量大，容易引发 XAML 编译问题；应分阶段调整并频繁构建验证。
- 批量深测与单站深测共用底层逻辑时，必须避免覆盖当前单站配置。
- 批量图表必须使用独立绑定属性，避免误显示成单站的最后一张图。
- 历史术语重命名应以新记录为准，不强行迁移旧历史数据。

## 7. 完成标准

当以下条件同时满足时，认为本轮 UI 重构完成：

- 核心构建与烟雾测试通过；
- 一级导航、单站、批量、网络复核、历史报告结构全部落地；
- 视觉风格统一为更清晰、更现代、更紧凑的 benchmark 风格；
- 用户可基于新流程完成“单站测试 → 批量对比 → 候选深测 → 异常复核”的完整闭环。
