# 大模型对话体验升级施工文档

> 日期：2026-05-01  
> 范围：`RelayBench.App` 大模型对话页、`RelayBench.Core` 聊天请求链路、聊天会话持久化、相关测试。  
> 目标：把当前大模型对话从“可用的测试聊天窗口”升级为“接近成熟 Web AI 对话产品，并保留 RelayBench 诊断特色”的高质量功能区。

---

## 1. 施工目标

本轮升级不只是堆按钮，而是围绕 4 个目标施工：

1. **效率更高：** 用户可以快速重试、复制、搜索、导出、拖拽附件。
2. **操作更顺：** 停止生成、编辑重发、快捷键、自动滚动都放在用户自然会找的位置。
3. **信息更清楚：** 每条回复展示协议、耗时、Token、错误原因和附件状态。
4. **视觉更精致：** 对话页交互要清晰、克制、大气，动画轻盈，不抢内容。

---

## 2. 当前能力基线

当前已经具备：

- 会话列表、新建、删除、自动标题。
- 编辑用户消息并从该位置重发。
- 预设：通用对话、代码助手、代码审查、总结整理、自定义保存。
- 图片和文本文件附件。
- 思考强度：自动、低、中、高。
- 多模型统一问答，最多 4 列并排。
- Markdown 渲染和代码块识别，支持复制代码块。
- OpenAI Chat、Responses、Anthropic Messages 自动探测与回退。
- 流式输出、停止命令、错误提示。

当前明显缺口：

- 无重新生成、整条消息复制、会话手动重命名、单独导出对话。
- 停止生成藏在右侧参数面板里，不适合流式输出时快速点击。
- 附件只能通过文件选择器添加，不支持拖拽。
- 会话和消息没有搜索。
- 快捷键、自动滚动、Token 用量、上下文窗口占比尚未建立。
- JSON / 结构化输出模式、PDF 附件、会话复制和分叉还未开放。

## 4. UI 施工方案

### 4.1 页面布局

维持当前三栏结构，但优化重心：

- **左侧会话栏：** 搜索框、会话列表、会话操作菜单。
- **中间对话区：** 消息、流式状态、停止按钮、底部输入区。
- **右侧参数抽屉：** 接口、模型、预设、系统提示词、多模型、生成参数。

调整原则：

- 中间对话区是主舞台，不再把关键运行操作放进右侧抽屉。
- 左侧会话栏只承载“找会话、切会话、管理会话”。
- 右侧抽屉只承载“参数配置”，打开后点击左侧空白区域仍可关闭。

### 4.2 顶部状态栏

顶部保留轻量状态：

- 左侧：当前模式，例如 `单模型：gpt-5.5` 或 `多模型 3 列`。
- 中部：状态摘要，例如 `Responses | TTFT 420 ms | 1,238 tokens`。
- 右侧：参数按钮。

流式输出时在顶部状态栏旁显示一个精致的小型运行状态：

- 文案：`正在生成`
- 图形：小型脉冲点或细条进度。
- 动画：透明度 0.55 到 1.0 轻微呼吸，周期约 1.2 秒。

### 4.3 左侧会话栏

新增元素：

- 搜索框：固定在会话列表上方。
- 每条会话卡片右侧悬浮菜单按钮。
- 会话选中态：蓝色边框，不改变会话顺序。
- 手动重命名后，标题旁显示轻量标记，表示不是自动标题。

会话卡片内容：

- 标题一行，超出省略。
- 更新时间 + 消息数量。
- 搜索命中时只过滤列表，不重排原有顺序。

### 4.4 中间对话区

消息气泡保持当前“用户右侧、助手左侧、互相错开”的模式。

新增消息工具条：

- 用户消息：编辑、复制、从此分叉。
- 助手消息：复制、重新生成、导出片段。
- 多模型回答：每列复制、每列重新生成、放大查看。

工具条展示方式：

- 默认隐藏。
- 鼠标悬停消息气泡时淡入。
- 键盘焦点进入消息时显示，保证可访问性。

### 4.5 输入区

底部输入区增加：

- 拖拽附件区域提示。
- 发送按钮旁固定停止按钮位置。
- 附件预览横向条。
- 快捷键提示用小字展示，不占主视觉。

输入区状态：

- 空闲：显示发送按钮。
- 生成中：发送按钮变灰，停止按钮高亮。
- 拖拽文件悬停：输入区边框变为蓝色，背景轻微变亮。
- 附件超限或格式不支持：在输入区上方显示短提示，不弹阻断窗口。

### 4.6 右侧参数抽屉

现有参数抽屉继续保留，但精修分组：

- 接口配置：Base URL、API Key、当前模型、拉取模型。
- 预设：应用、保存、删除。
- 生成参数：System Prompt、Temperature、Max Tokens、思考强度。
- 多模型对比：模型选择、已选模型、清空。
- 会话操作：清空、导出当前会话。

视觉要求：

- 抽屉宽度保持 420 px 左右。
- 分组使用轻量内嵌面板，不做过厚卡片堆叠。
- 重要操作使用主按钮，危险操作使用低饱和红色文字按钮。

## 5. 交互施工方案

### 5.1 重新生成

入口：

- 最后一条助手消息工具条显示「重新生成」。
- 右键菜单可补充同样入口。

行为：

1. 找到最后一条用户消息。
2. 删除该用户消息之后的所有消息。
3. 保留用户输入和附件。
4. 重新发送请求。
5. 新助手消息以流式方式生成。

边界：

- 如果最后没有用户消息，按钮禁用。
- 如果正在生成，按钮禁用。
- 如果最后是多模型回答，按当前多模型选择重新生成。

### 5.2 复制整条消息

入口：

- 消息工具条显示复制按钮。
- 多模型列内显示单列复制按钮。

复制内容：

- 普通消息复制 Markdown 原文。
- 多模型消息复制为 Markdown 分节：

```markdown
## model-a

...

## model-b

...
```

反馈：

- 按钮文字短暂变为「已复制」。
- 600 ms 后恢复。

### 5.3 会话重命名

入口：

- 会话卡片悬浮菜单。
- 会话标题双击进入编辑。

行为：

- 编辑时不改变会话位置。
- Enter 保存。
- Esc 取消。
- 空标题不保存，保留原标题。

数据：

- `ChatSessionSnapshot` 增加 `ManualTitle` 或 `IsTitlePinned`。
- 标题显示优先级：手动标题 > 首条用户消息自动标题 > 新会话。

### 5.4 停止按钮移入对话区

入口：

- 输入区右侧常驻「停止」按钮。
- 仅在 `IsChatStreaming == true` 时启用并高亮。

行为：

- 点击后立即取消当前 `CancellationTokenSource`。
- 当前助手消息显示「已停止生成」。
- 多模型模式下，对仍在生成的列逐列取消。

动画：

- 生成中停止按钮使用轻微红色脉冲边框。
- 停止成功后按钮回到普通禁用态，不弹窗。

### 5.5 导出对话

入口：

- 右侧参数抽屉「导出当前会话」。
- 会话卡片菜单「导出」。

格式：

- Markdown：保留角色、时间、模型、附件摘要、消息内容。
- TXT：纯文本可读版。

路径：

- 默认导出到 `RelayBenchPaths.ExportsDirectory/model-chat`。

命名：

```text
model-chat-YYYYMMDD-HHmmss-会话标题.md
model-chat-YYYYMMDD-HHmmss-会话标题.txt
```

### 5.6 拖拽添加附件

入口：

- 拖拽文件到输入区。
- 拖拽文件到中间对话区底部。

支持格式：

- 图片：`.png`、`.jpg`、`.jpeg`、`.webp`、`.gif`
- 文本：`.txt`、`.md`、`.cs`、`.json`、`.yaml`、`.yml`、`.xml`、`.xaml`、`.log`、`.csv`、`.ps1`

行为：

- 复用 `ChatAttachmentImportService`。
- 多文件逐个导入。
- 不支持的文件显示非阻断提示。
- 文件过大显示明确原因。

动画：

- 拖入区域时输入区出现 1 px 蓝色边框和浅蓝背景。
- 附件卡片加入时从 4 px 下方淡入，120 ms 完成。

### 5.7 会话搜索

入口：

- 左侧会话栏顶部搜索框。

搜索范围：

- 会话标题。
- 最近消息内容。

行为：

- 输入即过滤。
- 清空后恢复原列表顺序。
- 搜索不改变 `SelectedChatSession`，除非当前会话被删除。

### 5.8 快捷键

快捷键定义：

| 快捷键 | 行为 |
| --- | --- |
| `Ctrl+Enter` | 发送消息 |
| `Esc` | 生成中停止生成；编辑中取消编辑；参数抽屉打开时关闭抽屉 |
| `Ctrl+N` | 新建会话 |
| `Ctrl+F` | 聚焦会话搜索或当前对话搜索 |
| `Ctrl+Shift+E` | 导出当前会话 |

优先级：

1. 文本框内 `Ctrl+Enter` 发送。
2. `Esc` 优先处理当前最上层状态：抽屉 > 编辑 > 生成。
3. 不覆盖系统文本编辑快捷键。

### 5.9 自动滚动

行为：

- 默认开启。
- 流式输出时自动滚动到底部。
- 用户手动向上滚动后暂停自动滚动。
- 底部出现「回到底部」浮动按钮。
- 用户点击按钮后恢复自动滚动。

动画：

- 新消息进入时轻微淡入和上移。
- 回到底部按钮从底部淡入，不遮挡输入框。

## 6. 第二批功能细化

### 6.1 Token 用量统计

数据来源：

- 优先解析接口返回的 `usage`。
- 如果流式响应没有 usage，使用估算值。

模型：

- `ChatMessageMetrics` 增加：
  - `PromptTokens`
  - `CompletionTokens`
  - `TotalTokens`
  - `IsTokenCountEstimated`

展示：

- 消息底部显示：`1,238 tokens | 2.1 s | TTFT 420 ms | Responses`
- 估算值显示为 `约 1,238 tokens`。

### 6.2 消息内搜索

入口：

- 对话区顶部搜索按钮。
- `Ctrl+F` 触发。

行为：

- 当前会话内搜索。
- 展示命中数量。
- 上一个 / 下一个。

第一版实现：

- 先定位到命中消息。
- 命中文字高亮可放第二步做。

### 6.3 会话复制

入口：

- 会话卡片菜单「复制会话」。

行为：

- 复制当前会话的消息、参数、已选模型。
- 生成新 `SessionId`。
- 标题后缀加「副本」。
- 新会话插入列表顶部并选中。

### 6.4 对话分叉

入口：

- 任意消息工具条「从这里分叉」。

行为：

- 复制当前会话到新会话。
- 只保留分叉点及之前的消息。
- 新会话标题：`原标题 · 分叉`。
- 自动选中新会话。

边界：

- 分叉点是助手消息时，保留该助手消息。
- 分叉点是用户消息时，保留该用户消息，用户可以直接继续追问。

### 6.5 上下文窗口管理

数据来源：

- 优先使用模型上下文目录。
- 没有上下文信息时显示「未知」。
- Token 估算采用轻量策略，不阻塞发送。

展示：

- 输入区上方显示细条：
  - `上下文约 18%`
  - `接近上限` 时变为黄色。
  - `超过建议上限` 时变为红色。

行为：

- 接近上限只提醒，不禁止发送。
- 超过建议上限时发送前弹出确认。

## 7. 第三批功能细化

### 7.1 JSON / 结构化输出模式

入口：

- 参数抽屉新增「输出格式」。

选项：

- 普通文本。
- JSON 对象。
- JSON Schema。

协议映射：

- OpenAI Chat：使用 `response_format`。
- Responses：使用 `text.format` 或兼容字段。
- Anthropic：主要通过系统提示词约束，不强行写不兼容字段。

风险控制：

- 不同中转站对结构化字段支持差异较大。
- 默认不启用。
- 如果请求返回 400，错误提示要说明可能是结构化输出字段不兼容。

### 7.2 PDF 附件支持

第一版只做文本提取：

- 选择或拖拽 PDF。
- 提取文本。
- 作为文本附件发送。

限制：

- 不做扫描件 OCR。
- 不做版面还原。
- 不把整页渲染成图片发送。

提示：

- 如果 PDF 无可提取文本，提示用户该文件可能是扫描件。

## 8. 动画与视觉规范

### 8.1 总体风格

方向：清晰、轻盈、克制、精致。

不要做：

- 大面积闪烁。
- 持续晃动。
- 过强阴影。
- 每个按钮都放大。
- 高饱和渐变背景。

应该做：

- 轻微淡入。
- 小幅位移。
- 状态变化有节奏。
- 重要操作有明确反馈。
- 动画服务于信息，不抢内容。

### 8.2 动画参数

| 场景 | 动画 | 时长 | 缓动 |
| --- | --- | --- | --- |
| 新消息进入 | Opacity 0 → 1，Y 6 → 0 | 140 ms | CubicOut |
| 工具条出现 | Opacity 0 → 1 | 100 ms | QuadOut |
| 参数抽屉打开 | X 24 → 0，Opacity 0 → 1 | 180 ms | CubicOut |
| 参数抽屉关闭 | Opacity 1 → 0 | 120 ms | QuadIn |
| 附件加入 | Y 4 → 0，Opacity 0 → 1 | 120 ms | QuadOut |
| 拖拽悬停 | 边框颜色和背景色过渡 | 120 ms | Linear |
| 停止按钮生成态 | 边框透明度呼吸 | 1.2 s | SineInOut |
| 复制成功 | 文案切换 + 轻微背景闪光 | 600 ms | QuadOut |

### 8.3 状态反馈规范

- 成功：低饱和绿色，不用弹窗。
- 警告：琥珀色提示条。
- 错误：红色文本 + 可复制错误详情。
- 长任务：显示可取消状态。
- 非阻断问题：用输入区上方提示，不打断用户。

## 13. 推荐落地顺序

建议先做第一批，并按以下顺序执行：

1. 复制整条消息。
2. 重新生成。
3. 停止按钮移入输入区。
4. 会话重命名。
5. 会话搜索。
6. 导出对话。
7. 拖拽附件。
8. 快捷键和自动滚动。

理由：

- 复制和重新生成能快速建立消息操作模型。
- 停止按钮属于高频痛点，应尽早解决。
- 会话重命名和搜索属于左侧栏能力，可以独立施工。
- 导出和拖拽依赖较少，适合随后补齐。
- 快捷键和自动滚动涉及页面事件，放在第一批末尾统一接入更稳。

## 14. 风险控制

| 风险 | 控制方式 |
| --- | --- |
| 消息工具条遮挡正文 | 悬浮在气泡顶部右侧，正文留出最小 28 px 操作空间 |
| 自动滚动打断用户 | 用户上翻后暂停，显示回到底部按钮 |
| 重新生成删错消息 | 只从最后一条用户消息开始截断，并增加测试 |
| 会话搜索导致选中丢失 | 搜索只过滤显示，不改变活动会话 |
| Token usage 协议差异 | usage 缺失时回退估算值 |
| JSON 模式导致 400 | 默认关闭，错误提示指向结构化输出参数 |
| PDF 文本提取失败 | 明确提示扫描件不支持文本提取 |
