# RelayBench 仪表盘式工作台改造施工文档

> 版本：v1.1  
> 日期：2026-04-30  
> 状态：待审  
> 目标效果：参考用户提供的现代仪表盘截图，把 RelayBench 从“表单集合型工具”升级为“工作台 + 页面仪表盘”的视觉和交互结构。

## 1. 改造目标

本次改造的目标不是单纯调整字号、圆角或阴影，而是重建应用的第一层体验：

- 保留现有 WPF、MVVM、页面和业务逻辑。
- 将主窗口改造成清晰的工作台壳：顶部状态栏、左侧导航栏、中央页面区、底部状态条。
- 将「单站测试」先做成参考页，形成截图中类似的仪表盘结构：配置区、模式区、指标卡、图表区、实时日志区。
- 统一「大模型对话」「批量评测」「应用接入」「网络复核」「历史报告」的页面头部、卡片、表格、图表和滚动条风格。
- 保证窗口化、全屏、高 DPI 下文字不挤压、不重叠，滚动条稳定显示为圆柱样式。

## 2. 视觉方向

### 2.1 整体气质

目标风格为“轻量运营仪表盘”：

- 背景：浅蓝灰渐变或低饱和浅色，不使用大面积深色背景。
- 面板：白色或近白色卡片，1px 边框，轻阴影。
- 主色：以蓝色为主，少量紫蓝渐变用于主按钮、Logo 和进度条。
- 字体：保持 `Microsoft YaHei UI`，正文 12-13px，页面标题 16-18px。
- 信息密度：比普通网页更紧凑，但比现在更有呼吸感。
- 动效：只保留 hover、press、展开、下拉、进度条等必要微交互。

### 2.2 关键视觉元素

| 元素 | 目标效果 | WPF 落点 |
|------|----------|----------|
| 顶部栏 | 左侧品牌，中间任务进度胶囊，右侧窗口按钮 | `MainWindow.xaml` 顶部 Row |
| 左侧导航 | 白色圆角导航卡，选中项蓝色渐变，左侧指示条 | `NavigationRailBorderStyle`、`NavigationListBoxItemStyle` |
| 内容页头 | 页面标题、说明、右侧状态/时间 | 新增 `WorkbenchPageHeaderStyle` |
| 业务卡片 | 白底、轻阴影、统一内边距 | `SectionPanelBorderStyle` 拓展 |
| 指标卡 | 4 列 KPI，数字突出，状态色区分 | 新增 `MetricCardStyle` |
| 图表卡 | 白底图表，支持扫光/loading | 复用现有图表控件与 `LiveChartSweepBorderStyle` |
| 日志面板 | 深色控制台风格，等宽字体，状态着色 | 新增 `ConsolePanelStyle` |
| 底部状态 | 当前状态 + 上次运行时间 | `MainWindow.xaml` 底部状态条 |

## 3. 范围边界

- 不引入 WebView 做整体 UI。

## 5. Shell 布局方案

### 5.1 主窗口结构

推荐结构如下：

```text
MainShellBorder
├─ TitleBar，48px
│  ├─ Logo + RelayBench 标题
│  ├─ GlobalTaskProgress 胶囊
│  └─ 最小化 / 最大化 / 关闭
└─ WorkbenchBody
   ├─ LeftNavigationRail，固定宽度 220-236px
   └─ ContentWorkspace
      ├─ PageHeader，高度自适应
      ├─ PageContent，滚动或局部滚动
      └─ StatusBar，高度自适应
```

实现要点：

- `MainWindow.xaml` 中保留现有 `WindowChrome` 和标题栏拖拽逻辑。
- 主体区域使用 `Grid`，左侧导航固定宽度，右侧内容 `*`。
- 右侧页面内部不再随意塞满所有内容，而是统一使用“页面头 + 页面体 + 状态条”。
- 页面体需要明确 `MinHeight=0`，避免窗口化时 ScrollViewer 只剩一小截。

### 5.2 顶部栏

顶部栏保持 48px 左右：

- Logo 使用蓝紫渐变圆角方块，内部显示 `R` 或现有图标。
- 标题显示 `RelayBench 接口测试工作台`。
- 全局任务进度使用胶囊控件，显示任务名、进度条、状态文本。
- 窗口按钮保持右侧，hover 状态轻背景，关闭按钮 hover 变红。

注意：

- 顶部栏的任务进度不要占满标题栏，窗口较窄时允许文本省略。
- 进度条区域必须 `IsHitTestVisible=False`，不影响窗口拖拽。

### 5.3 左侧导航栏

左侧导航栏做成独立白色卡片：

- 宽度建议 `220px`，最小不低于 `204px`。
- 外层使用 20-22px 圆角、1px 边框、轻阴影。
- 导航项高度 42-46px。
- 选中项使用浅蓝渐变背景、蓝色文字、左侧竖向指示条。
- 底部保留关于/版本/项目链接区域。

导航项不要在切换页面时移动位置。选中哪个，蓝框只移动到对应项，列表本身不重新排序。

## 6. 页面结构方案

### 6.1 页面头部

所有页面统一使用页面头：

```text
PageHeader
├─ 左侧：页面标题 + 页面说明
└─ 右侧：当前状态 / 上次运行 / 快捷按钮
```

示例：

- 单站测试：标题「单站测试」，说明「配置接口地址与模型，选择测试模式后一键运行」。
- 大模型对话：标题「大模型对话」，说明「模拟真实对话，支持文件、图片、代码块与多模型对比」。
- 批量评测：标题「批量评测」，说明「同时测试多个接口或模型，生成对比结果」。

### 6.2 页面体滚动策略

为避免“窗口化后滚动条只剩一小截”的问题，页面体采用以下规则：

- 外层 `ContentWorkspace` 控制整体高度。
- 每个页面根节点使用 `Grid`，关键行设置为 `Height="*"`。
- 需要滚动的区域使用一个主 `ScrollViewer`，避免 ScrollViewer 嵌套 ScrollViewer。
- 聊天页这类复杂页面允许局部滚动，但每个滚动区必须有明确的 `MinHeight=0` 和所在 Grid 的 `RowDefinition Height="*"`。
- 所有 `ScrollViewer` 统一使用 `WorkbenchScrollViewerStyle`。

## 7. 单站测试参考页施工

单站测试作为第一参考页，按截图效果改成仪表盘布局。

### 7.1 推荐布局

```text
单站测试页面
├─ 配置卡片
│  ├─ 接口地址
│  ├─ API Key
│  ├─ 模型
│  └─ 拉取模型 / 开始测试 / 停止
├─ 测试模式卡片
│  ├─ 快速测试
│  ├─ 深度测试
│  ├─ 兼容性检测
│  └─ 流式输出检测
├─ KPI 指标区，4 列
│  ├─ 成功率
│  ├─ 平均延迟
│  ├─ Token/s
│  └─ 协议支持
├─ 下方双列
│  ├─ 图表卡：延迟/吞吐/成功率
│  └─ 实时日志卡：深色控制台
└─ 页面状态条
```

### 7.2 配置卡片

配置卡片目标：

- 一眼能看到 URL、Key、模型和主操作。
- 模型输入框右侧保留拉取按钮。
- 历史接口入口可以做成右侧按钮或下拉菜单，不占用太多纵向空间。

WPF 建议：

- 使用 `Grid` 做 4-5 列布局。
- URL 列宽最大，Key 和模型次之，操作按钮固定宽。
- 窗口低于 1280px 时允许按钮换行到下一行。

### 7.3 KPI 指标卡

新增 `MetricCardStyle`：

- 高度 88-104px。
- 顶部小标签，底部大数字。
- 状态色：成功绿色、警告橙色、错误红色、信息蓝色。
- 支持 `TextTrimming`，避免长模型名撑开布局。

建议绑定：

- `SuccessRateDisplay`
- `AverageLatencyDisplay`
- `TokensPerSecondDisplay`
- `ProtocolSupportDisplay`

如果当前 ViewModel 暂时没有这些聚合字段，第一版可以使用已有测试结果字段组合展示，不新增复杂统计逻辑。

### 7.4 图表与日志

图表卡：

- 延续现有图表渲染能力。
- 外层使用统一卡片样式。
- loading 时使用现有扫光动画。

日志卡：

- 背景 `#0F172A`。
- 字体 `Cascadia Code` 或 `Consolas`。
- 时间戳灰色，成功绿色，警告黄色，错误红色，普通信息蓝色。
- 高度不要固定死，窗口化时参与剩余空间分配。

## 8. 大模型对话页施工

大模型对话页重点是释放中间聊天区域，同时保留多模型对比能力。

### 8.1 布局

```text
大模型对话
├─ 顶部工具条
│  ├─ 当前模型 / 多模型选择摘要
│  ├─ 新建会话
│  ├─ 预设
│  └─ 参数按钮
├─ 主体三栏
│  ├─ 会话列表，左侧，宽 220-260px
│  ├─ 聊天区，中间，占主要空间
│  └─ 状态/指标栏，右侧，可折叠
└─ 输入区，固定在聊天区底部
```

### 8.2 参数入口

接口地址、API Key、模型、系统提示词、对比模型、思考强度、采样参数放入右上角参数菜单：

- 点击参数按钮打开右侧抽屉或浮层。
- 点击关闭按钮关闭。
- 点击聊天区空白处关闭。
- 点击会话列表项不关闭、不改变会话顺序，只移动选中蓝框。

### 8.3 消息样式

- 用户消息靠右。
- 大模型回复靠左。
- 双方气泡错开，不占满整行。
- Markdown 使用 WPF 原生渲染控件。
- 代码块必须识别语言、使用深色背景、支持复制按钮。
- 文件和图片附件显示为小标签或缩略卡。

### 8.4 多模型对比

多模型统一问答按列展示：

- 选择 2-4 个模型时，发送同一段输入。
- 回答区生成 2-4 列，每列一个模型。
- 每列顶部显示模型名、状态、耗时、Token。
- 流式输出时各列独立追加内容。
- 单模型时回到普通气泡模式。

窗口化时：

- 2 列可保持并排。
- 3-4 列在窄窗口下横向滚动，不挤压到不可读。

## 11. 响应式与窗口化规则

必须重点验证窗口化，因为当前多次问题都出现在窗口化状态。

规则：

- 主窗口最小尺寸继续使用 `MinWidth=1120`、`MinHeight=700`。
- 页面根 `Grid` 的滚动区所在行必须是 `Height="*"`。
- 不在可变高度区域内使用无约束的 `StackPanel` 包住大型内容。
- 表格、聊天区、日志区所在父容器设置 `MinHeight="0"`。
- 右侧参数抽屉最大宽度 420px，最小宽度 340px。
- 多模型 3-4 列时优先横向滚动，不把列压到低于 280px。
- KPI 卡在 1440px 以上 4 列，在 1120-1280px 可降为 2 列。

## 14. 风险与处理

| 风险 | 表现 | 处理方式 |
|------|------|----------|
| XAML 体量大 | `MainWindow.xaml` 已承载大量 Overlay 和 Shell 逻辑 | 优先只改结构和样式，不移动业务绑定 |
| ScrollViewer 嵌套 | 窗口化时滚动条变成一小截 | 每页只保留主滚动区，局部滚动区必须有明确高度约束 |
| 阴影性能 | DropShadowEffect 过多导致低端机器卡顿 | 阴影只给一级卡片和浮层，列表项不加重阴影 |

| 卡片过多 | 页面看起来碎片化 | 卡片只用于功能块，不在卡片内继续套卡片 |
| 文案乱码 | 发布包或 GitHub 页面中文异常 | 所有 XAML 和 Markdown 使用 UTF-8，提交前检查中文显示 |

|------|------|---------|

### 16.2 统一样式

新增以下通用组件样式：

- `DashboardToolbarBorderStyle`：页面内工具条外壳。
- `DashboardToolbarButtonStyle`：普通工具按钮，如「历史接口」「查看图表」。
- `DashboardToolbarPrimaryButtonStyle`：页面主操作按钮，如「开始测试」。
- `DashboardIconButtonStyle`：小图标按钮，如模型输入框右侧的拉取模型。

- `DashboardMenuPanelStyle`：右上角菜单、参数抽屉和二级菜单面板。
- `DashboardOverlayPanelStyle`：居中弹窗外壳。
- `DashboardActionCardStyle`：应用卡、工具卡、报告卡。
- `DashboardCompactTableStyle`：候选列表、深测结果、历史接口列表。

### 16.3 文案与图标规则

- 主操作使用“动词 + 对象”，如「开始测试」「拉取模型」「导出报告」。
- 二级菜单按钮保持短词，不写长说明，如「历史接口」「图表」「参数」「预设」。
- 危险操作用红色描边或红色文字，如「清空记录」「删除会话」。
- 可逆但有影响的操作使用确认弹窗，如「还原默认」「清空会话」。
- 所有按钮保留 `ToolTip`，但页面上不额外写大段说明。

## 26. 复查后建议采用的更优 UI 改法

### 26.1 把“按钮堆”改成“页面命令区”

很多页面现在容易出现一排按钮：历史接口、查看图表、开始测试、停止测试、拉取模型。建议统一为：

```text
页面命令区
├─ 左侧：当前上下文摘要
├─ 中间：次级工具按钮
└─ 右侧：主操作按钮
```

这样「主操作」永远在右侧，用户不会在一堆按钮里找。

### 26.2 把“高级内容”收进抽屉

适合收进右侧抽屉的内容：

- 单站能力模型。
- 单站并发阈值说明。
- 大模型对话参数。
- 应用接入写入预览。
- 网络复核高级参数。

这样主页面会更像截图里的仪表盘：首屏只放高频信息，低频配置从右侧打开。

### 26.3 图表卡统一 hover 操作

现在有多个「放大图表」按钮。更好的做法：

- 图表卡右上角固定一个放大图标。
- 鼠标 hover 图表时显示「查看大图」浮层按钮。
- 所有图表都进同一个图表总览弹窗。

这样页面视觉更干净。

### 26.4 网络复核做成工具卡分组

网络复核功能很多，建议按分组展示：

- 基础复核：基础网络、网页 API。
- 性能复核：测速、路由 / MTR。
- 出口复核：IP 与分流、IP 风险。
- 连接复核：NAT / STUN、端口扫描。

每组用小标题 + 工具卡，选中工具卡后右侧切换配置和结果。比长条子菜单更现代，也更接近仪表盘。

### 26.5 应用接入做成“软件资产卡”

应用接入的应用列表建议像资产面板：

- 卡片顶部：软件图标、名称、协议。
- 卡片中部：当前入口、配置来源、可用状态。
- 卡片底部：Trace、还原、应用状态。

这样比文本列表更直观，也更适合展示 Codex、Claude CLI、OpenAI 兼容客户端等不同软件。

### 26.6 历史报告做成时间线

历史报告建议从普通列表改成时间线列表：

- 左侧显示日期和排序。
- 中间显示运行标题、类型标签、摘要。
- 右侧显示状态或导出标记。

选中后右侧详情保持卡片式。这样历史记录会更像专业诊断工具，而不是简单文本堆。

### 26.7 大模型对话采用“OpenWebUI 式紧凑工作台”

大模型对话建议采用：

- 左侧会话列表固定。
- 中间消息区最大化。
- 参数、预设、多模型放右侧抽屉。
- 输入区 sticky 在底部。
- 多模型回答用列视图。

这个方案比三栏常驻更省空间，尤其窗口化时更舒服。

### 26.8 减少纯文字按钮，增加图标按钮

适合改成图标按钮的入口：

- 拉取模型。
- 放大图表。
- 原始 Trace。
- 删除附件。
- 删除已选模型。
- 复制代码。
- 打开 GitHub。

注意：图标按钮必须有 tooltip，危险操作仍要有明确确认。
